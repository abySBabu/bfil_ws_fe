1. Dashboard, Navigation Menu, Avatar Menu

 

 

Image 1.1: Navigation menu (left) and avatar menu (top right)

 

The navigation menu on the left side of the screen is used to switch between the various sections of the web app. It is always available from any of the sections. The sections available in the navigation menu depend on the role of the user.

 

The avatar menu can be accessed by clicking on the avatar icon with the user’s initials in them on the top right of the screen. It displays the user’s name and role on the top followed by buttons for the:

● My profile page

● Language toggle (English/Kannada)

● Logout

 

 

 

Image 1.2: Navigation Menu for super admin role where all sections are available

 

The dashboard page is the first page seen by the user on sign in. It has a map view and and information cards for

● Key impact indicators

● Supply side interventions

● Demand side interventions

 

The key impact indicator cards show the total values of the key indicators from all supply and demand side activities in the watershed(s) managed by the user.

Each of these cards have a graph button on the bottom right of the card that can be clicked to see graphical representation of key indicators from completed activities over time.

 

The supply and demand side intervention cards show the total value of all activities, for each intervention type, in the watershed(s) managed by the user.

 

The map shows the locations of the supply and demand side interventions in the watershed(s) managed by the user. The map can be expanded to be viewed in fullscreen and has various zoom levels to view location pins of the activities.

 

 

 

 

 

 

 

 

3. User Management

 

 

Image 3.1: User management page

 

The user management page contains the list of all users. New users can be added and existing users can be edited or blocked/deleted in this page by the administrator.

The super admin also has the option to change a user’s password here.

 

To create a new user, click on the ‘Add User’ button on the top right corner of the page. This will open a dialog with fields for the necessary data to be filled in. Mandatory fields are marked with an asterisk (*) at the end of their names. Once all mandatory fields are filled, the submit button will be enabled allowing the user to add a new user.

 

 

Image 3.2: User edit dialog

To edit a user, click on the edit button for that user. This will open a dialog with fields of the selected user’s data filled in. The user can edit the values of the fields from here. Certain fields are not editable and will be greyed out. If all mandatory fields are filled, the user can click on the update button to save their edits.

 

To block a user, click on the block button for that user. This will open a dialog asking the user to confirm that they want to block the selected user.

 

If a user has been blocked, unblock and delete buttons will show up in the place of the block button. Clicking on either of these buttons will open a dialog asking the user to confirm if they want to unblock or permanently delete the blocked user.

(Note: A user can only be deleted if they don’t have any activities assigned to them, see image 3.4)

 

To change a user’s password, click on the password button for that user (Note: This function is only available for the super admin). This will open a dialog asking the super admin user to enter and reenter the new password they wish to set for the selected user.

4. Role Management

 

 

Image 4.1: Role management screen

 

The role management screen contains the list of all roles. Roles that are assigned to users in user management are created, edited, or deleted here. The screen permissions for a role can be managed here when it is edited or created. The following roles are present by default:

● Community resource person

● Project manager

● Technical officer

● Program officer

● State project head

● Chief manager head office

 

To add a role, click on the ‘Add Role’ button on the top right corner of the page. This will open a dialog where you can create a new role by entering its name and description and choosing which permissions it has. (Note: A role should have at least one permission)

 

To edit a role, click on the edit button of that role. This will open a dialog with the values of the selected role’s name, description, and permissions which can be changed from here (Note: The selected role must have at least one permission or the user won’t be able to save their edits).

 

 

 

To delete a role, click on the delete button of that role. This will open a dialog asking the user to confirm that they want to delete the selected role (Note: The user can only delete a role if it has not been assigned to any user).

7. Watershed Mapping

 

 

Image 7.1: Watershed mapping page

 

The watershed mapping page contains the list of users and the watersheds they are mapped to. Users only see activities from the watershed mapped to them in the dashboard and watershed activity pages.

Watershed mappings can be added or edited in this page.

 

To add a mapping, click on the ‘Add Mapping’ button in the top right corner of the page. This will open a dialog to add a new mapping. Select a user and the watershed(s) you wish to map to them (Note: CRP users can have only one watershed mapped to them).

Click the add button once you have selected all necessary details to create a new mapping.

 

To edit a mapping, click on the edit button for that mapping. This will open a dialog with the selected mapping’s details filled in. You can edit the watershed(s) mapped from here. Once you have made edits, click on the update button to save your changes.

8. Watershed Activity

 

 

Image 8.1: Watershed activity page

 

The watershed activity page contains the list of all activities from the watershed(s) mapped to the user. The user can add, edit, view, and approve/reject activities in this page.

 

To add an activity, click on the ‘Add Activity’ button on the top right corner of the page. This will open a dialog to create a new activity where you can fill in details about the new activity. Once all mandatory fields are filled, the add button will be enabled allowing you to create a new activity.

 

To edit an activity, click on the edit button for that activity (You can only edit an activity created by you). This will open a dialog with the selected activity’s details filled in, you can change these values from here. Once all mandatory fields are filled, the update button will be enabled allowing you to save your changes. When an activity is edited, the remarks of that edit are recorded to the history table where all who have access to that activity can see them.

 

To view an activity in detail, click on the view button for that activity. This will open a dialog with all the activity’s details and update history. This dialog has no functionality like adding or editing and is only to be used to view the activity’s details and history.

The history table (see image 8.4) of an activity contains the remark history of each time that activity was edited, approved, or rejected along.

 

 

 

To approve or reject an activity, click on the process button for that activity (Users can only process activities whose status matches their approver status). This will open a dialog that has the same content as the detail view dialog, but will have a field for remarks and the options to approve (send to the next approver level) or reject (send to the previous approver level). When an activity is approved or rejected, the remarks of that process are recorded to the history table where all who have access to that activity can see them.

10. Report

 

 

Image 10.1: Report page

 

The report page contains the various reports for activities. The report page is divided by tabs into the report types: watershed report, donor report, activity report, and map location.

 

The reports for all tabs except map location can be filtered by year and activity report they can be filtered by activity type too.

 

All tabs except map location also have the options to download the report by clicking the download button or print the report by clicking the PDF button. Both of these buttons are located at the right side above the report table.

 

 

Image 10.2: Map tab in report page

 

The map tab allows the user to see activities on the map. The user can filter the activities they want to view by status and activity type.
